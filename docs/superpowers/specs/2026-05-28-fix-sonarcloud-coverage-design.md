# Fix SonarCloud Coverage Reporting

**Date**: 2026-05-28
**Status**: Approved

## Problem

SonarCloud quality gate shows "0.0% Coverage on New Code" on all PRs. The `sonar-project.properties` on `main` does not configure `sonar.javascript.lcov.reportPaths`, so SonarCloud has no coverage data to report.

## Root Cause

- `ci.yml` runs `npm run test:coverage` (generates `coverage/lcov.info`) but does **not** upload it to SonarCloud
- `sonar-project.properties` lacks coverage path configuration
- SonarCloud automatic analysis (GitHub app) does not execute tests — it only reads coverage data if explicitly uploaded via `sonarqube-scan-action`

## Solution

Integrate SonarCloud scan into the existing `ci.yml` — no separate workflow.

### Changes

#### 1. `ci.yml` — Add SonarCloud scan step

Add a step to the existing `Tests & Coverage` job, after `npm run test:coverage`:

```yaml
- name: SonarCloud Scan
  uses: SonarSource/sonarqube-scan-action@689fb39b34b9aa95ebc5f8f119343ddd51542402
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
    SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
```

This reuses the `coverage/lcov.info` already generated by the preceding `npm run test:coverage` step. No duplicate test execution.

#### 2. `sonar-project.properties` — Add coverage config

```
sonar.javascript.lcov.reportPaths=coverage/lcov.info
sonar.exclusions=src/**/*.test.ts,src/**/*.test.tsx,src/**/*.spec.ts,src/**/*.spec.tsx,src/test/**,coverage/**,node_modules/**
sonar.tests=src
sonar.test.inclusions=src/**/*.test.ts,src/**/*.test.tsx,src/**/*.spec.ts,src/**/*.spec.tsx
```

#### 3. `package.json` — Add `tmp` override

Add `"tmp": "^0.2.6"` to `overrides` (security fix for CVE-2024-27980).

#### 4. `package-lock.json` — Lockfile update

Auto-generated from `npm install` for `tmp` resolution.

## What Is NOT Changed

- No separate `sonarcloud.yml` workflow
- No duplicate test runs
- Existing `ci.yml` structure preserved

## Verification

After merge, verify on next PR that SonarCloud quality gate shows coverage > 0%.
